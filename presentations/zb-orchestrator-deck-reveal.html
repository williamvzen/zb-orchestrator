Feature: zb-agent

  • Discovers projects under ~/zb-projects.
  • Intent: fuzzy match or --reason (LLM picks category/name from catalog).
  • --ticket / ZB_AGENT_TICKET injects Jira workflow into the seed.
  • --ide-only · --cli-only · --rebase-main · --install → ~/.local/bin.
  • Multi-repo: ~/.zb-workspaces/<names>.code-workspace — Agent uses first project.

zb-agent — seed prompt

def build_cursor_agent_prompt(...):
    if ticket:
        lines.append(f"Work on Jira ticket {ticket}.")
    lines.append(f"Intent: {intent}")
    if ticket:
        lines.append(
            "Workflow: ... init-local-ticket-branch ... commit-it-then ..."
        )
    else:
        lines.append(
            "Workflow: ... Jira ticket → init-local-ticket-branch + commit-it-then; "
            "else commit-it-then with [NOTICKET]"
        )
    return "\n".join(lines)

Feature: timesheet-agent

  • Default: two parallel cursor agent --print runs — Jira MCP + calendar JSON from screenshot.
  • Screenshot cached under .calendar_vision_cache/; strict JSON → Python normalizes → format_calendar_meetings.py.
  • Output markers: [JIRA_AGENT_OUTPUT] and [TIMESHEET-OUTPUT].
  • Flags: --no-jira, --capture (macOS), --image, --json, --with-times.

Demo script (5–10 min)

  1. zb-agent: ./zb-agent.py --list → open project → show Intent in Agent.
  2. Ticket: ./zb-agent.py … --ticket ECOMM-XXXX --no-open → show seed.
  3. timesheet: ./timesheet-agent -i cal.png --no-jira (or full parallel with MCP).
  4. Optional: timesheet-ui upload.
  5. Close: generate-timesheet SKILL — agent must call tools.

Closing

zb-orchestrator treats Cursor as the automation engine: Python orchestrates when and where agents run; Skills and Rules define how — yielding repeatable, auditable workflows from a small local codebase.
